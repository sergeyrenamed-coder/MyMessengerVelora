# Hybrid Messenger v8 — localhost MVP+++

Локальный браузерный мессенджер с логикой ближе к Telegram Desktop, но в твоём тёмном стиле.

## Что есть в v8

- регистрация и вход по `email / username + пароль`
- разные аккаунты в разных вкладках через `sessionStorage`
- личные чаты, группы и **каналы**
- E2EE для текста и файлов в текущей браузерной сессии
- очистка кэша дешифрования при скрытии вкладки и выходе
- отправка файлов
- голосовые сообщения с удержанием микрофона
- **кружки / video notes** с круглыми bubble в чате
- ответ на сообщение
- редактирование своего текстового сообщения
  - через контекстное меню по **ПКМ**
  - через **double click** по своему текстовому сообщению
- пересылка текстового сообщения
- удаление своего сообщения
- профиль/информация о чате открываются **по центру** и сделаны выше, чтобы меньше скроллить
- в канале скрыта кнопка звонка
- настройки в стиле Telegram:
  - профиль
  - устройства
  - данные и память
  - emoji / sticker / premium section
  - **темы**
- редактирование профиля через центрированную карточку
  - аватар меняется **по нажатию на аватар**
- устройства показываются в читаемом виде без сырого `Mozilla/5.0 ...`
- панель `Эмодзи / Стикеры / GIF` открывается **справа**, как drawer
- **паки** для emoji и stickers
- preview окна пака по клику на стикер/emoji в чате
- premium emoji сейчас работает как **free beta**
- hover-меню на скрепке: фото/видео, документ, опрос, геопозиция, кошелёк
- клик по скрепке открывает обычный проводник для отправки файла
- **базовые WebRTC звонки**:
  - аудио
  - видео
  - mute микрофона
  - выключение камеры
  - демонстрация экрана
- Enter = отправить, Shift + Enter = новая строка
- скрытые scrollbar-полосы в боковых панелях и модалках

## Требования

- Node.js **22+**
- npm

Для Windows лучше распаковывать проект **не в OneDrive**, а в обычную папку вроде:

```text
C:\Projects\hybrid-messenger-localhost
```

## Запуск

```bash
npm install
npm start
```

После этого открой:

```text
http://localhost:3000
```

## Как тестировать два аккаунта сразу

### Вариант 1 — в двух вкладках

1. Открой первую вкладку: `http://localhost:3000`
2. Открой вторую вкладку:

```text
http://localhost:3000/?fresh=1
```

Во второй вкладке токен будет очищен, и можно войти под другим аккаунтом.

### Вариант 2 — другой браузер / приватное окно

Тоже работает.

## Как протестировать звонки

1. Войди в **два разных аккаунта** в двух вкладках или двух браузерах.
2. Открой общий личный чат или группу.
3. Открой профиль/инфо чата и нажми кнопку звонка.
4. В окне звонка выбери `Аудио` или `Видео`.
5. Во второй вкладке подтверди входящий звонок.

Для screen share используй кнопку `Экран` уже во время звонка.

## Ограничения текущей версии

- E2EE всё ещё рассчитан на одну доверенную браузерную сессию
- это локальный MVP+, а не production-ready Telegram clone
- звонки реализованы как **базовый WebRTC для localhost / малых комнат**, без SFU/MCU
- качество и стабильность записи голосовых/кружков зависят от браузера и `MediaRecorder`
- пересылка сейчас включена только для **текстовых** сообщений
- GIF-панель использует ссылки на GIF как контент сообщения
- custom image emoji и стикеры — локальная кастомизация конкретного аккаунта/браузера
- паки и темы хранятся локально в браузере пользователя
- полноценная multi-device синхронизация ключей ещё не реализована
